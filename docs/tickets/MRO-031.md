# MRO-031: Client deployment + config pipelines

## Description
Implement per-client Kubernetes resource pipelines. For each MatrixClientView: Deployment (nginx image, port 80), ConfigMap (config.json via @switch on client name for Element vs Cinny config shapes), Service (ClusterIP port 80), HTTPRoute (client subdomain). Use @switch to differentiate Element Web and Cinny configuration formats.

## Acceptance Criteria
- [ ] Element Deployment created with correct nginx image and config mount
- [ ] Cinny Deployment created with correct nginx image and config mount
- [ ] Element config.json has Element-specific shape
- [ ] Cinny config.json has Cinny-specific shape
- [ ] ClusterIP Services created on port 80
- [ ] HTTPRoutes created for element.{domain} and cinny.{domain}
- [ ] element.{domain} serves Element Web
- [ ] cinny.{domain} serves Cinny
- [ ] Both clients can log in to Synapse

## Notes
The @switch operator selects the correct config template based on client name, keeping a single pipeline for all client types.
