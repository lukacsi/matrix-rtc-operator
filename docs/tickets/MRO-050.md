# MRO-050: Matrix status writers (fan-in)

## Description
Implement matrix-status-writers.yaml: fan-in status from individual components back to parent CRs. Synapse/CNPG/client/JWT status -> Views -> MatrixStack -> MatrixRTCStack. Use Patcher target for status fields. @definedOr for uninitialized fields.

## Acceptance Criteria
- [ ] Synapse status propagated to MatrixStack
- [ ] CNPG status propagated to MatrixStack
- [ ] Client status propagated to MatrixStack
- [ ] JWT service status propagated to MatrixStack
- [ ] MatrixStack status propagated to MatrixRTCStack
- [ ] `kubectl get matrixrtcstack` shows accurate ready/not-ready with component breakdown
- [ ] @definedOr handles uninitialized fields gracefully

## Notes
Status fan-in is essential for operational visibility. Each component writes its status independently, and the fan-in aggregates to parent resources.
