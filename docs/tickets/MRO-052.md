# MRO-052: NetworkPolicy templates

## Description
Implement NetworkPolicy resources for all components. Default deny ingress per namespace. Explicit allow rules for expected traffic flows. Gated by networkPolicy.enabled flag in the CR spec.

## Acceptance Criteria
- [ ] Default deny ingress NetworkPolicy per namespace
- [ ] Explicit allow rules for component-to-component traffic
- [ ] NetworkPolicies gated by networkPolicy.enabled
- [ ] With networkPolicy.enabled=true, only expected traffic flows
- [ ] With networkPolicy.enabled=false, no NetworkPolicies created

## Notes
Be aware of kube-router issues with NetworkPolicy (see homelab notes). Test with a CNI that properly supports NetworkPolicy (Cilium).
