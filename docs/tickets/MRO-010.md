# MRO-010: Enterprise values.yaml with @param comments

## Description
Create values.yaml following Bitnami patterns with @param comments for auto-generated README parameter tables. The file should be organized into clear sections: global, matrix, livekit, stunner, certManager, infrastructure. Include security contexts, resource presets (nano/small/medium/large), probe defaults.

This enables consistent documentation generation and provides a self-documenting configuration surface for operators deploying the chart.

## Acceptance Criteria
- [ ] values.yaml renders successfully with `helm template`
- [ ] All sections documented with @param annotations
- [ ] Security contexts default to restricted (non-root, read-only rootfs, drop all capabilities)
- [ ] Resource presets defined: nano, small, medium, large
- [ ] Probe defaults (liveness, readiness, startup) included for all components
- [ ] Sections present: global, matrix, livekit, stunner, certManager, infrastructure

## Notes
- Follow Bitnami @param comment format: `## @param <path> <description>`
- Resource presets follow Bitnami pattern: `resourcesPreset` string selects from predefined CPU/memory combos, `resources` object allows full override.
- Global section should include `global.imageRegistry`, `global.imagePullSecrets`, `global.storageClass`.
