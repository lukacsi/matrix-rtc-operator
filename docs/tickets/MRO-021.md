# MRO-021: Synapse deployment pipeline

## Description
Implement the synapse-deployment controller in matrix-views-to-k8s.yaml. Container spec: Synapse image, ports 8008/8448, volume mounts (homeserver.yaml ConfigMap, data PVC, media PVC). Use @hash annotation from Secret data for rollout triggers. Security context: non-root, read-only rootfs.

## Acceptance Criteria
- [ ] Synapse Deployment created from MatrixServerView
- [ ] Container exposes ports 8008 and 8448
- [ ] homeserver.yaml mounted from ConfigMap
- [ ] Data and media PVCs mounted
- [ ] @hash annotation triggers rollout on Secret changes
- [ ] Security context: non-root, read-only rootfs
- [ ] Synapse pod starts and serves /_matrix/client/versions

## Notes
Volume mounts need careful ordering. The @hash pattern ensures pods restart when secrets change without exposing secret values in annotations.
