# MRO-014: Adapt LiveKit pipelines -- gateway-agnostic

## Description
Port the 3 existing LiveKit pipeline files from livekit-declarative-operator, removing Envoy Gateway specifics and using generic gatewayRef from RTCInfrastructure. The pipelines should work with any Gateway API implementation (Traefik, Envoy, Cilium, etc.).

## Acceptance Criteria
- [ ] `livekit-to-views.yaml` adapted from source repo
- [ ] `livekit-views-to-k8s.yaml` adapted from source repo
- [ ] `livekit-status-writers.yaml` adapted from source repo
- [ ] HTTPRoute parentRefs use `infra.httpGateway` from RTCInfrastructure (not hardcoded Envoy)
- [ ] No Envoy Gateway-specific resources remain (no EnvoyProxy, EnvoyPatchPolicy, etc.)
- [ ] Existing livekit-config-generator pattern preserved
- [ ] `infra.yaml` adapted: remove infra-to-envoy-* controllers, keep STUNner + cert-manager controllers

## Notes
- Source files: `~/AI/Projects/livekit-declarative-operator/charts/livekit-operator/templates/operator/`
- Key files to port: `livekit-to-views.yaml`, `livekit-views-to-k8s.yaml`, `livekit-status-writers.yaml`, `infra.yaml`
- The gateway-agnostic approach means the operator only creates standard Gateway API resources (HTTPRoute, UDPRoute) and references a user-provided Gateway via RTCInfrastructure.
- STUNner integration stays as-is since it's inherently gateway-agnostic (uses its own GatewayClass).
