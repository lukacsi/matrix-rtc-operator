# MRO-017: GitHub Actions CI

## Description
Set up a GitHub Actions CI workflow that validates the Helm chart on every push to master and on pull requests. The workflow should run helm lint, helm template, and chart-testing (ct lint) to catch configuration errors and template rendering failures early.

## Acceptance Criteria
- [ ] `.github/workflows/ci.yaml` exists and runs on push to master and PRs
- [ ] Workflow runs `helm lint` against the chart
- [ ] Workflow runs `helm template` to verify rendering
- [ ] Workflow runs `ct lint` (chart-testing) for comprehensive chart validation
- [ ] CI passes on a clean repo with no changes
- [ ] CI catches values.yaml errors and template rendering failures
- [ ] Uses pinned action versions (not @latest)

## Notes
- Use `helm/chart-testing-action` for ct lint setup.
- Use `azure/setup-helm@v4` for Helm installation.
- ct lint requires a `ct.yaml` config file — include it in the repo root or chart directory.
- Consider adding a matrix strategy for multiple Kubernetes versions in the future, but keep it simple for now.
