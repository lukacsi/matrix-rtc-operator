# MRO-032: Security contexts for client pods

## Description
Harden client pod security contexts. Set runAsNonRoot, readOnlyRootFilesystem, drop ALL capabilities. nginx requires writable /tmp and /var/cache/nginx directories — mount emptyDir volumes for these paths.

## Acceptance Criteria
- [ ] Client pods run as non-root
- [ ] readOnlyRootFilesystem enabled
- [ ] All capabilities dropped
- [ ] emptyDir volumes mounted at /tmp and /var/cache/nginx
- [ ] nginx serves content correctly with restricted security context

## Notes
Standard nginx hardening pattern. The emptyDir volumes satisfy nginx's runtime write requirements without compromising the read-only rootfs policy.
