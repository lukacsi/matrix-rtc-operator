# MRO-101: FR-3: Pipeline testing framework — dctl test

## Description
Propose a `dctl test` CLI command for unit testing dcontroller pipelines. Input CRs (YAML files) -> run pipeline -> assert output resources. Support unit test format, snapshot testing, and CI-friendly output. This is the #1 adoption barrier for DevOps teams.

## Acceptance Criteria
- [ ] Issue filed on dcontroller repo
- [ ] Proposed API and test format examples included
- [ ] CI integration approach described
- [ ] Snapshot testing workflow described

## Notes
Without testability, DevOps teams can't adopt dcontroller in production. A `dctl test` command would enable TDD-style pipeline development.
