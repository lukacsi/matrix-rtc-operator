# MRO-200: Tiered media storage (SSD->HDD archival)

## Description
Implement tiered media storage for Synapse. SSD (local-path) for active media with off-hours CronJob that rsyncs older media to HDD (zfs-nfs) for archival.

## Acceptance Criteria
- [ ] Active media stored on SSD (local-path StorageClass)
- [ ] CronJob rsyncs media older than threshold to HDD (zfs-nfs)
- [ ] CronJob runs during off-hours
- [ ] Synapse can still serve archived media

## Notes
HDD only for bulk static media; SSD for everything else (silence priority per user preference).
