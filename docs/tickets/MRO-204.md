# MRO-204: Matrix bridges (Discord, Telegram, Signal)

## Description
Declarative bridge management for Matrix. A Bridge CRD that deploys mautrix-* bridge pods (Discord, Telegram, Signal) with configuration derived from the MatrixStack.

## Acceptance Criteria
- [ ] Bridge CRD defined for declarative bridge configuration
- [ ] mautrix-discord deployment from Bridge CR
- [ ] mautrix-telegram deployment from Bridge CR
- [ ] mautrix-signal deployment from Bridge CR
- [ ] Bridges auto-register with Synapse
- [ ] Bridge enable/disable via CR spec

## Notes
Each mautrix bridge has its own config format and registration requirements. The Bridge CRD should abstract these differences.
