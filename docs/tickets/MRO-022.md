# MRO-022: homeserver.yaml generation via @concat

## Description
Implement the synapse-config controller that uses @concat to build homeserver.yaml in a ConfigMap's stringData field. Static fields: server_name, public_baseurl, experimental_features (msc3266/4140/4222), rate limits, listeners. Dynamic fields: database connection (@cond for managed vs external mode), domain, registration setting. Join with Secret for DB credentials via DecodeSecretData.

## Acceptance Criteria
- [ ] ConfigMap created with valid homeserver.yaml in stringData
- [ ] Static config: server_name, public_baseurl, experimental_features, rate limits, listeners
- [ ] Dynamic config: database connection string varies by managed/external mode
- [ ] DB credentials injected via DecodeSecretData
- [ ] @cond correctly switches between managed and external DB config
- [ ] Generated homeserver.yaml is valid YAML
- [ ] Synapse starts successfully with generated config

## Notes
Follow the livekit-config-generator pattern from thesis (views-to-k8s.yaml line 102). The @concat approach builds the config as a string template with interpolated values.
