# MRO-020: matrix-to-views.yaml — server + networking views

## Description
Create the view layer that decomposes MatrixStack into focused views for downstream controllers. MatrixStack -> MatrixServerView + MatrixNetworkingView. Use @join with RTCInfrastructure for auto-discovery via @definedOr. Server view captures: synapse image, resources, DB config, MSC flags. Networking view captures: subdomains, gateway refs, domain.

## Acceptance Criteria
- [ ] Apply MatrixStack CR
- [ ] MatrixServerView created with correct fields (image, resources, DB config, MSC flags)
- [ ] MatrixNetworkingView created with correct fields (subdomains, gateway refs, domain)
- [ ] @join with RTCInfrastructure works via @definedOr for auto-discovery

## Notes
This is the foundational view layer for the entire Matrix pipeline. All Phase 2-4 controllers depend on these views.
