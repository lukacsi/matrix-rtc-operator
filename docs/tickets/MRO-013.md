# MRO-013: RBAC templates -- least privilege

## Description
Create ClusterRole with explicit permissions per apiGroup/resource/verb for all resources the operator watches and creates. ServiceAccount with automountServiceAccountToken: false. Follow least-privilege principle — only grant what the operator actually needs.

## Acceptance Criteria
- [ ] ClusterRole with explicit rules (no wildcards on verbs or resources)
- [ ] ServiceAccount with `automountServiceAccountToken: false`
- [ ] ClusterRoleBinding connecting ServiceAccount to ClusterRole
- [ ] Operator can create all target resources: Deployments, StatefulSets, Services, Secrets, ConfigMaps, HTTPRoutes, UDPRoutes, CNPG Clusters, Certificates, ClusterIssuers
- [ ] Operator cannot access resources outside its declared scope
- [ ] RBAC templates gated by `rbac.create` and `serviceAccount.create` toggles

## Notes
Required API groups and resources:

| apiGroup | Resources | Verbs |
|----------|-----------|-------|
| apps | deployments, statefulsets | get, list, watch, create, update, patch, delete |
| "" (core) | services, secrets, configmaps | get, list, watch, create, update, patch, delete |
| gateway.networking.k8s.io | httproutes | get, list, watch, create, update, patch, delete |
| stunner.l7mp.io | udproutes | get, list, watch, create, update, patch, delete |
| postgresql.cnpg.io | clusters | get, list, watch, create, update, patch, delete |
| cert-manager.io | certificates, clusterissuers | get, list, watch, create, update, patch, delete |
| lukacsi.org | * (all CRDs + views) | get, list, watch, create, update, patch, delete |

- `lukacsi.org/*` is acceptable since the operator owns the entire API group.
- Consider splitting into Role (namespaced) + ClusterRole (cluster-scoped) if RTCInfrastructure is the only cluster-scoped resource.
