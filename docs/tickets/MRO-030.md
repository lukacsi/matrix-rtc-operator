# MRO-030: matrix-to-client-views controller (@unwind)

## Description
Add client view generation to matrix-to-views.yaml. Build a clients array from MatrixStack spec, use @unwind to create one MatrixClientView per enabled client, and @select to filter out disabled clients. Each view carries the client name, image, subdomain, and config.

## Acceptance Criteria
- [ ] Enable Element + Cinny -> 2 MatrixClientViews created
- [ ] Disable Cinny -> only Element MatrixClientView exists
- [ ] Re-enable Cinny -> Cinny MatrixClientView reappears
- [ ] Each view contains correct client name, image, subdomain, and config

## Notes
The @unwind operator fans out a single MatrixStack into N client views, enabling per-client downstream pipelines without duplication.
