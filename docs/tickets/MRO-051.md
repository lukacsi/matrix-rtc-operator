# MRO-051: Probes for all components

## Description
Add liveness, readiness, and startup probes to all component deployments. Synapse: liveness /_synapse/health, readiness /_matrix/client/versions. LiveKit: :7880. Redis: redis-cli ping. lk-jwt-service: :8080/healthz. Element/Cinny: HTTP GET /.

## Acceptance Criteria
- [ ] Synapse: liveness probe on /_synapse/health
- [ ] Synapse: readiness probe on /_matrix/client/versions
- [ ] LiveKit: probe on :7880
- [ ] Redis: probe via redis-cli ping
- [ ] lk-jwt-service: probe on :8080/healthz
- [ ] Element: HTTP GET / probe
- [ ] Cinny: HTTP GET / probe
- [ ] All pods have all 3 probes (liveness, readiness, startup)

## Notes
Startup probes prevent premature restarts during slow initialization (especially Synapse with large databases).
