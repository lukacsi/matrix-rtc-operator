# MRO-011: values.schema.json validation

## Description
Create a JSON Schema (values.schema.json) for values.yaml that provides validation at `helm install`/`helm upgrade` time. The schema should enforce conditional validation (e.g., database.external requires host/port when mode=external) and enum constraints for all mode fields.

This catches misconfigurations early with clear error messages instead of failing at runtime with cryptic template errors.

## Acceptance Criteria
- [ ] `helm lint` catches missing required values with descriptive errors
- [ ] `helm install` with invalid mode value fails with clear error message
- [ ] Conditional validation: database.external.host required when database.mode=external
- [ ] Enum constraints for all mode fields (e.g., database.mode: [internal, external])
- [ ] Schema covers all top-level sections from values.yaml

## Notes
- Helm natively supports values.schema.json — no plugins needed.
- Use JSON Schema draft-07 `if`/`then` for conditional validation.
- Keep schema in sync with values.yaml — consider adding a CI check for drift.
