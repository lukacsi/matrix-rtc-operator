# MRO-023: CNPG Cluster pipeline (conditional)

## Description
Conditionally create a CNPG PostgreSQL Cluster CR based on database.mode. Use @select on database.mode == "managed" to gate creation. Create postgresql.cnpg.io/v1 Cluster CR with fields: instances, storage, storageClass, enablePDB, resources. CNPG auto-generates credentials in a Secret. When mode=external, pipeline emits empty set and CNPG Cluster is deleted.

## Acceptance Criteria
- [ ] database.mode="managed" -> CNPG Cluster CR created
- [ ] CNPG Cluster has correct instances, storage, storageClass, enablePDB, resources
- [ ] CNPG auto-generates credentials Secret
- [ ] Database becomes ready
- [ ] database.mode="external" -> no CNPG Cluster exists
- [ ] Synapse uses external connection string when mode=external
- [ ] Switching from managed to external deletes the CNPG Cluster

## Notes
CNPG single-instance PDB blocks node drains. Use enablePDB: false in Cluster spec for single-instance deployments.
