# MRO-001: Test CNPG Cluster CRD creation via dcontroller

## Description
Verify dcontroller can create third-party CRDs (postgresql.cnpg.io/v1 Cluster). This is a prerequisite for the matrix-rtc-operator, which needs to manage CNPG PostgreSQL clusters as part of the Synapse deployment pipeline.

## Acceptance Criteria
- [x] Confirmed generic target support in dcontroller Resource struct
- [x] Documented RBAC requirements for third-party CRD targets

## Notes
The dcontroller Resource struct is fully generic — any apiGroup/kind combination works as a pipeline target. The key constraint is RBAC: ClusterRole rules must explicitly enumerate each target resource (apiGroups, resources, verbs).

## Resolution
CLEAR — Resource struct fully generic, any apiGroup/kind. RBAC ClusterRole rules must enumerate target resources. See docs/dcontroller-assessment.md.
